5、部署到Github

这一步会出很多问题，一定要及时处理然后将问题补充到这个笔记中！！

1.创建github仓库

​ 注意！！！

在博客的目录下有一个_config.yml文件，对其进行修改

4.安装插件

这些都是在E:\Blogger\my_blog

这个生成的文件里去做的（不知道是不是会有影响）

自动将生成的静态网页推送到git仓库

为什么需要这个插件？

  1. 自动化部署：替代手动复制文件到仓库的操作
  2. 版本控制：自动保留每次生成的版本记录
  3. 持续集成：可配合 GitHub Actions 实现自动构建部署
  4. 多平台支持：适用于 GitHub/GitLab/Gitee 等平台
